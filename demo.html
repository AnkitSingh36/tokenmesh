Stage 1 — Chunker

  • Splits at sentence boundaries
  • 40-word windows (default)
  • overlap=0 (no inflation)
  • Preserves full sentences

Stage 2 — Dedup (TF-IDF)

  • Cosine similarity matrix
  • Removes near-duplicate chunks
  • Keeps longer (richer) chunk
  • Neural in real library

Stage 3 — Scorer

  • Runs only if query is set
  • Scores chunks vs query
  • Drops irrelevant chunks
  • Query-aware compression
